## 一、项目概述

### 1.1 产品定位

RenderUI 是一个面向车载嵌入式系统的**高性能控件渲染框架**，采用现代 C++ 开发，基于 Wayland/Weston 显示协议，使用 OpenGL 硬件加速渲染。

### 1.2 设计目标

- **简洁易用**：API 设计参考 Qt 风格，但更轻量
- **代码可读性优先**：智能指针、现代 C++、避免复杂模板和宏
- **分层清晰**：各层级内部消化问题，不向上抛异常
- **适度扩展**：留有重构空间，避免过度设计
- **易于调试**：完善的日志和调试工具链
- **高性能**：GPU 加速、异步流水线、双缓冲优化

### 1.3 技术约束

#### 基础技术栈（阶段一至三）
```
核心语言：C++17/20
构建系统：CMake + Ninja
显示协议：Wayland/Weston
渲染引擎：OpenGL ES 3.0+
2D 绘图：Cairo（阶段一）、Skia（阶段四）
信号槽：CppSoft_Signals (VDK 实现)
JSON: nlohmann/json
日志：plog
枚举反射：magic_enum
模型加载：tinyobjloader / tinyply
图像加载：stb_image
IPC: DBus
```

#### 扩展技术栈（阶段四起）
```
网络通信：muduo
本地存储：LevelDB
3D 模型：assimp
单元测试：GoogleTest
性能分析：Perfetto Trace
```

## 三、非功能需求

### 3.1 性能指标

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 首屏渲染时间 | < 200ms | 启动到首帧显示 |
| UI 响应延迟 | < 16ms (60fps) | 触摸到视觉反馈 |
| 内存占用 | < 200MB (基础) | RSS 常驻内存 |
| 帧率稳定性 | 60fps ±2% | 连续运行 1 小时 |

### 3.2 编码规范

- **代码风格**: Google C++ Style
- **C++ 标准**: C++17/20
- **智能指针**: `std::unique_ptr`, `std::shared_ptr`
- **宏使用**: 禁止复杂宏，仅允许简单常量定义
- **头文件保护**: `#pragma once`
- **继承深度**: 不超过 3 层
- **模板复杂度**: 仅使用易理解的泛型
- **注释语言**: 中文
- **命名规范**: 参考 Qt 风格（如 `QPushButton`, `addWidget()`）

### 3.3 设计原则

#### 推崇（优先级从高到低）
1. 简洁易用
2. 代码可读性
3. 基础能力完备
4. 适度扩展空间
5. 分层清晰
6. 易于调试
7. 高性能

#### 抵制
1. 庞大复杂的架构
2. 设计冗余
3. 陈旧技术栈
4. 晦涩语法
5. 庞大的第三方库
6. 严重耦合

## 五、Demo 场景定义

### 5.1 车载全景 360 影像 Demo

**场景描述**: 模拟车载环视系统，展示框架综合能力

**功能需求**:
1. 4 路摄像头画面拼接（视频流渲染）
2. 3D 车身模型叠加（3D 渲染）
3. 触摸切换视角（事件系统）
4. 动态轨迹线绘制（Cairo/Skia 绘制）
5. 车速/档位显示（文本/图标）
6. DBus 接收车辆信号
7. 多视图切换（正常/俯视/侧视）

**技术验证点**:
- 视频流低延迟渲染
- 2D/3D混合渲染
- 高帧率稳定性
- 信号驱动 UI 更新
